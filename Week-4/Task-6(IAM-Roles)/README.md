# Task 6: IAM Roles

## Objective

Configure Identity and Access Management (IAM) roles to provide secure access to Azure resources and enable Azure Data Factory to interact with Azure Blob Storage.

## Steps Performed

1. Opened the **Access Control (IAM)** section of the Storage Account.
2. Added the required role assignment.
3. Assigned the **Storage Blob Data Contributor** role.
4. Verified that the role assignment was successfully created.
5. Enabled Azure Data Factory to access Blob Storage resources.

## Components Used

* Azure Storage Account
* Access Control (IAM)
* Azure Data Factory
* Storage Blob Data Contributor Role

## Purpose

IAM roles are used to control access to Azure resources. The Storage Blob Data Contributor role allows Azure Data Factory to read, write, and manage files stored in Azure Blob Storage.

## Outcome

Successfully assigned the required role and provided Azure Data Factory access to Blob Storage.
